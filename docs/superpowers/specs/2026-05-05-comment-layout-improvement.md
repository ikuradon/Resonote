# デザイン設計書: コメントレイアウトの改善 (A案)

Flowタブにおけるコメント行の情報過多と要素の重なりを解消し、視認性と操作性を向上させます。

## 1. 目的

- 長いユーザー名やメタデータ（再生位置、投稿時間）によるリアクションボタンとの重なりを解消する。
- 1行あたりの情報密度を下げ、ゆとりのあるレイアウトにする。
- モバイル環境での誤操作（誤タップ）を防止する。

## 2. 変更内容 (A案: リアクションボタンの下部移動 + ヘッダー整理)

### 2.1 ヘッダーの整理 (`CommentCard.svelte`)

- **リアクションボタンの移動**: 現在ヘッダー右側に配置されているリアクションボタン一式を、コメント内容の下（フッター領域）へ移動します。
- **ユーザー名の省略表示**: ユーザー名表示部に `min-width: 0` と `text-overflow: ellipsis` を適用し、非常に長い名前でも他の要素（投稿時間など）を押し出さないようにします。
- **配置の最適化**: ヘッダーは左側に「アバター + 名前 + NIP-05」、右側に「投稿時間」を配置するシンプルな構成に変更します。

### 2.2 フッター（アクション行）の新設

- **新レイアウト**: コメント内容の下に、新しいアクション行を追加します。
- **再生位置の配置**: 「Flow」モード特有の再生位置ボタン（例: 01:23）を、このアクション行の左端に配置します。
- **リアクションボタンの配置**: 返信、リポスト、リアクション、メニューボタンをアクション行の右側に十分な間隔（gap）を持って配置します。

## 3. 視覚的構成案 (Flowモード時)

```
[Avatar] [Username (省略あり)] [✓NIP-05]        [Time]
------------------------------------------------------
[Comment Content...                                  ]
[...                                                 ]
------------------------------------------------------
[Position]              [Reply] [Repost] [Like] [Menu]
```

## 4. 影響範囲

- `src/lib/components/CommentCard.svelte`: 主な変更対象。
- `CommentList.svelte`: `CommentCard` を呼び出しているが、Propsの変更はないため影響は最小限。

## 5. テスト計画

- **表示確認**: 長いユーザー名を含むコメントで重なりが発生しないことを確認。
- **レスポンシブ確認**: モバイル幅（350px程度）でアクションボタンが押しやすく配置されていることを確認。
- **機能確認**: リアクションボタン、再生位置ボタンのクリックイベントが正しく発火することを確認。

## 6. 特記事項

- 「Shout」モード（既にボタンが下部にある）との一貫性を高めるため、共通のレイアウトパーツとして整理します。
